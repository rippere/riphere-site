# riphere.com Redesign — Design References

Curated from `emmabostian/developer-portfolios` (24.5k★, ~1,791 sites), filtered to
Ben's niche, 16 visited, 4 finalists rendered in a real browser to verify craft.
Staged for the redesign — feeds the aesthetic-direction / color / type decisions.

> **One-line takeaway:** The best of these treat the **portfolio as a build artifact,
> not a brochure** — tall single-column editorial, a **serif display voice paired with
> monospace technical labels**, restrained on a dark *or* warm-neutral canvas with **one
> electric accent** and near-zero gratuitous motion. Credibility is *engineered*
> (identity-as-code, achievements-as-receipts), not asserted.

---

## Ranked shortlist

### ⭐ Top 4 — model these most closely

**1. Marcus Sánchez — https://marcuss.pro** — *the closest positioning match*
- "Builds with AI agents and still owns every line that ships" ≈ Ben's "researcher who ships."
- **Warm off-white/cream** editorial canvas (NOT dark), giant **serif display name** + navy ink, **monospace** metadata labels, numbered sections (01–06), section headers written as shell commands (`git log --author=marcus`, `tail -f /now`). Embedded **"ask me anything" AI agent that answers in his voice**; formal/fun tone toggle; hard metrics ("1,001 merged PRs").
- **Steal:** serif-display + mono-label system · shell-command section headers · in-page "ask my AI" agent as live proof you build agents.

**2. Maxime Haegeman — https://www.maximehaegeman.com** — *best dark/serious direction*
- "From messy data to reliable predictions, at scale" — research-to-production framing.
- True near-black + single **acid-lime accent**; **serif headline** against **monospace** everywhere; hero is a literal **`maxime.py` code-editor card** (`class Maxime(DataScientist, DataEngineer)`); section labels as system output (`STATUS: ACTIVE // SKILLS`); live "System Status: Online" footer.
- **Steal:** dark + one electric accent + serif/mono · the "identity-as-typed-code-object" hero · system-status framing of mundane sections.

**3. Maciej Pulikowski — https://pulik.dev** — *the receipts/credibility blueprint*
- Security researcher (AI + security); serious-AI adjacent.
- Dark, centered single column. Standout: **research rendered as structured receipt rows** — each finding with vendor logo, a **severity badge**, an explicit **reward amount ($10,000…)**, and a write-up link.
- **Steal:** render shipped systems as verifiable receipt-rows (*system · metric badge · outcome · proof link*) — the CRM, bias-agent tool, fMRI scorer become scannable third-party-validated proof instead of prose.

**4. Peter Steinberger — https://steipete.me** — *the thinking-in-public IA*
- Respected AI-tools builder; positions via shipping evidence + writing.
- Austere, blog-centric, anti-portfolio: avatar → tagline → reverse-chron posts. Authority from dated, specific writing, not visuals.
- **Steal:** a first-class writing/notes stream so the "researcher" half reads as ongoing thinking-in-public. (Borrow the IA, bring your own visual craft.)

### Also strong

**5. Gowtham Sridhar — https://www.gowthamsridhar.com** — "AI engineer, HCI Scientist" at a research institute; literal scientist-who-ships. *Steal:* research credentials (ResearchGate/papers) side-by-side with shipped systems + an **AI-assistant-readable "quick facts" block**.

**6. Ahmed Tokyo — https://ahmedtokyo.com** — CTO with a dedicated **"Academia"** nav item beside Projects. *Steal:* an explicit **Research/Academia** nav peer to Work — structurally signals "researcher who ships" before a word is read.

**7. Luca Félix — https://luca-felix.com** — "thinks like a designer / cares how things work as much as how they look." *Steal:* the one-line dual-identity hook (the *sentence pattern*, e.g. "a researcher who ships…").

**8. Toukoum (Raphaël Giraud) — https://toukoum.fr** — the canonical **"the portfolio IS a chat with my AI"** pattern. *Steal:* the conversational-portfolio mechanic. ⚠️ Tone is playful/light — **opposite** of Ben's register. Take the mechanic; render it dark, restrained, credible (more like #1's embedded agent than this memoji version).

## What this surfaces for Ben's redesign (decisions to make)

1. **Canvas fork — dark-serious vs. warm-editorial.** Both read "serious + design-forward."
   #2/#3 are dark; #1 (the closest positioning match) is warm cream. *Real choice for the
   aesthetic-direction round.*
2. **Type system — serif display + monospace labels** is the dominant credibility move across
   the whole set (vs. the current site's system-sans). Strong candidate direction.
3. **Identity-as-code** — hero framed as a typed class / shell commands / system-status lines.
   Uniquely *earned* by an agent-builder; risks cliché if overdone.
4. **Achievements-as-receipts** — the 4 flagship systems as scannable rows (metric badge ·
   outcome · proof link), not paragraphs. Directly serves the "verifiable → inbound" goal.
5. **A writing/thinking stream** as a peer to Work — the home for the learn-in-public engine's
   output; makes the "researcher" half legible.
6. **★ Signature move available only to Ben:** a live **"ask my agent"** element that *is* the
   proof he builds agents — done in a serious register (#1), not playful (#8). Potential
   defining element of the site.
